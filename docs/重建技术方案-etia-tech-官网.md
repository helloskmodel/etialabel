# 怡天科技官网(www.etia-tech.com)重建技术方案

- **文档日期**:2026-07-02
- **配套文档**:`docs/审计报告-etia-tech-官网.md`
- **前提**:域名与企业主体不变,**沿用现有 ICP 备案**;重建仅换代码与部署,不涉及重新备案。

---

## 1. 重建范围

| 模块 | 现状 | 重建后 |
|---|---|---|
| 前台展示层 | 未提供代码(疑似 SPA) | **Nuxt 3(SSR)**,利于国内搜索引擎收录与首屏 |
| 后台 CMS | Vue 2.5 + Element-UI(EOL) | **Vue 3 + Vite + Element Plus** |
| 后端 API | Lumen 7 + PHP 7(EOL) | **Laravel 11 + PHP 8.3** |
| 数据库 | MySQL | 沿用现有表结构,迁移数据 |
| 对象存储 | 腾讯云 COS | 沿用(密钥需轮换) |
| 部署 | 旧 Docker + 失效镜像源 | 现代化 Docker + CI/CD |

内容模块沿用现有划分:首页、标签行业应用、全系产品、设备行业应用、新闻中心、关于怡天、服务支持(下载/故障/证书)、留言发信。

---

## 2. 技术选型与理由

| 层 | 选型 | 理由 |
|---|---|---|
| 前台 | **Nuxt 3 + Vue 3** | 企业官网重 SEO,SSR/SSG 首屏与收录优于纯 SPA;生态活跃、长期维护 |
| 后台 | **Vue 3 + Vite + Element Plus** | 与现有后台同源,界面逻辑可大量复用;Vite 构建快 |
| API | **Laravel 11 + PHP 8.3** | 现有代码已是 Laravel/Lumen 分层(Controller/Service/Model),平移成本低;在支持周期内 |
| DB | **MySQL 8**,结构沿用 | 迁移风险最低,内容零丢失 |
| 部署 | Docker + 国内云 + CI/CD | 替换失效的淘宝镜像源与过旧 nginx:1.13 |

> 备选:后台若想与前台统一为一套技术,也可用 Nuxt 全栈;但考虑复用现有后台逻辑,建议后台维持 Vue3+ElementPlus。

## 3. 分阶段计划与工期估算

> 工期以"理想工程工时"计,单人全职推进的粗估;实际随设计稿是否就绪、联调配合速度浮动。**关键前置见第 5 节。**

| 阶段 | 内容 | 估算 | 依赖 |
|---|---|---|---|
| **阶段 0:安全止血** | 轮换密钥、关 APP_DEBUG、移除 .env、登录/发信限流 | **1–2 天** | 可立即做,独立于重建 |
| **阶段 1:盘点与准备** | 数据库结构梳理、内容盘点、接口清单、环境与账号确认 | **3–5 天** | 云账号、DB 访问 |
| **阶段 2:API 重写** | Laravel 11 骨架 + 各模块接口平移 + 安全加固(bcrypt、参数化 SQL、格式校验、限流) | **2–3 周** | 阶段 1 |
| **阶段 3:后台 CMS 重写** | Vue3 + Element Plus,复用现有界面逻辑,对接新 API | **2–3 周** | 阶段 2 接口就绪 |
| **阶段 4:前台展示层** | Nuxt 3 SSR,按设计稿实现各页面,对接 API,SEO 优化 | **2–3 周** | **设计稿 / 视觉稿** |
| **阶段 5:数据迁移 + 联调** | 迁移脚本、内容搬迁、端到端联调 | **约 1 周** | 阶段 2–4 |
| **阶段 6:测试 / 加固 / 灰度上线** | 回归测试、安全复检、灰度切换、监控 | **约 1 周** | 全部 |

**整体估算:约 8–11 周(全量重建)。**
若前台暂缓、先做 API + 后台,则约 5–7 周。

## 4. 安全加固(重建时一并解决,对应审计报告)

- 密码存储:`md5` → **bcrypt / argon2**
- 批量更新:拼接式 SQL → **参数绑定**
- 发信接口:加 **格式校验 + 验证码 + 限流**
- 登录:加 **失败锁定 / 限流**
- 配置:`.env` 移出代码库,生产 `APP_DEBUG=false`
- 依赖 / 镜像:全部升级到维护期内版本,替换失效镜像源
