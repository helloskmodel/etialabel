# ETIA Label 官网 · 服务承诺与协作约定
# ETIA Label Website — Service Commitment & Working Agreement

> 目的:结束反复来回、避免浪费,让每一步都对齐、可验证。
> Purpose: end the back-and-forth, stop the waste, make every step aligned and verifiable.

---

## 我的承诺 · My commitments

### 1. 唯一事实来源 · Single source of truth
你的每一份行业资料,先存成仓库文件(`_docs/sources/<行业>.md`),我**严格按文件建站**。
不在文件里的内容,我不建。
> Every brief you give me is first saved as a repo file. I build strictly from that file. If it is not in a file, it does not go on the site.

### 2. 绝不编造 · Never invent
只用你资料里的真实型号与参数。任何缺口一律标注
「Contact ETIA / 待核实」,**绝不用我猜的数据填补**。
> Only real models and specs from your material. Any gap is flagged "Contact ETIA / to be verified" — never filled with a guess.

### 3. 视觉参考走截图 · Visual references by screenshot
本构建环境**打不开任何外部网站**(Computype、etiatech.com 均被网络策略挡住)。
所以:你发**截图 + logo 文件**,我照着做;不靠文字描述猜设计。
> This build environment cannot open any external website. So you send screenshots + the logo file, and I match them — I do not guess a design from words.

### 4. 先做，后改 · Build first, then correct
我先产出**真实页面**给你看,你对着实物改。
不再用「文字描述 ↔ 我猜」的方式空对空。
> I produce a real page first; you react to the real thing. No more design-by-description ping-pong.

### 5. 一个稳定网址，永远最新 · One live URL, always latest
我们把部署理顺:一个 Vercel 网址永远显示开发分支最新版。
**你评判的永远是最新版,不是旧部署。**
> We fix deployment so one Vercel URL always shows the newest build. You always judge the latest, never a stale deploy.

### 6. 省 TOKEN · Token discipline
回复简短;能做就做,不拿选择题烦你。
只有「猜错的代价 > 问一句」时我才问。
> Short replies; I act instead of asking. I ask only when a wrong guess would cost more than the question.

### 7. 集中在一个窗口 · One window
构建与决策都在**这个窗口**落地(存成文件)。
其它窗口可以帮你搜集参考,但结论要以文件形式带回这里,否则跨窗口就丢。
> Build and decisions land in THIS window as files. Other windows may gather references, but conclusions must come back here as files, or they are lost across windows.

## 每个行业「做完」的标准 · Definition of done (per industry)
1. 原始资料已存成 `_docs/sources/<行业>.md`(你确认无误)。
2. 数据 JSON 的型号/参数与该文件**逐条一致**(我贴出来给你核)。
3. 页面构建通过 `validate.py`:0 断链、0 中文泄漏、温度分离正确。
4. 已推送,且在稳定网址上能看到。
5. 缺口都带「待核实」标注,清单列给你。

> Done = source saved · JSON matches the source line-by-line · validator clean · pushed & visible on the live URL · gaps flagged.
